# Gamma MCP Proposal Generation Instructions
**Hey AI Assistant!** When the user asks you to "Generate a Gamma proposal for [Folder Name]", follow these precise instructions to ensure the presentation inherits the Paioneers brand identity and the context the user provided.

## Step 1: Read the Context
1. Go to `proposal_deck/workspace/[Folder Name]`.
2. Read the `reason.txt` file to understand the prospect, their pains, the proposed solution, and commercials.
3. Read the images inside `screenshots/` using your vision capabilities. Extract any relevant context (e.g., current architecture flaws, prospect's existing website, competitive landscape).

## Step 2: Read the Brand Identity
1. Read `Presentation_Templates copy/brand/brand.tokens.json` to get the core color palette.
   - Primary: `#C96442`
   - Secondary / Ink: `#262624`
   - Backgrounds: `#F6F2EF`, `#FFFFFF`
2. Read `Presentation_Templates copy/brand/brand.css` to grasp the typography (Arial, sans-serif) and the design geometry (rounded corners, soft shadows).

## Step 3: Call the Gamma MCP
Use your Gamma MCP integration to generate the presentation. Incorporate the following constraints into your generation prompt for Gamma:

### A. Narrative & Structure
- **Length**: 5-8 slides.
- **Tone**: Professional, authoritative, yet approachable (The Paioneers Voice).
- **Structure**:
  1. **Cover Slide** (Title, prospect name, date)
  2. **The Current Situation** (Insights from screenshots and `reason.txt` problem statement)
  3. **The Paioneers Approach** (Our proposed solution mechanics)
  4. **Scope & Delivery** (Timeline, what we build)
  5. **Commercials** (Investment summary from `reason.txt`)
  6. **Call to Action / Next Steps**

### B. Visual & Theme Instructions for Gamma
Tell Gamma to use the following styling:
- **Theme/Colors**: Create a custom theme using the hex codes: Primary `#C96442`, Dark Text `#262624`, Light Background `#F6F2EF`.
- **Fonts**: Clean Sans-Serif (Arial or similar).
- **Images**: Do not use generic, cheesy stock photos. Use abstract, high-quality, minimal 3D shapes or structural architecture images to express the concepts, or rely purely on typography and the provided screenshots if possible.
- **Layouts**: Use wide, airy layouts with strong typography hierarchy (big distinct headers, small concise body text). Avoid walls of text. Use bullet points or 2-column grids for comparisons (e.g., "Current State vs. Future State").

## Step 4: Finalize
Once the Gamma presentation is generated by the MCP, reply to the user with the presentation link.
